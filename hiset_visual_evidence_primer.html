Reading visual evidence before you answer.

Charts, graphs, tables, diagrams, maps, and political cartoons all show up on the test. The habits below apply to every one of them.

1
Before You Answer Anything
Two habits that apply no matter what kind of visual you're looking at — and that prevent most careless mistakes before they happen.
Read the frame before the data▾
Key concepts
Title states the subjectAxis labels state what's measuredUnits change the meaningLegend explains color-coding
Resources
Khan AcademyDisplaying and describing data — overview unit↗
Test tip: Spend the first few seconds reading only the title, axis labels, and legend — before looking at any individual bar, point, or slice. Answering before doing this is the single most common cause of careless mistakes.
Don't assume the scale▾
Key concepts
Check what each gridline equalsGraphs that don't start at zeroEqual gaps ≠ equal amountsBroken-axis symbols
Video resources
YouTubeMisleading line graphs — Khan Academy↗
Test tip: Find at least two labeled gridlines before reading any value off a chart. Never assume gridlines automatically count by 1, 5, or 10 — confirm it from the numbers actually printed on the axis.
2
Reading Each Chart Type
Bar graphs, line graphs, pie charts, tables, and diagrams each reward a slightly different reading strategy. Here's what to look for in each.
Bar graphs — comparing categories▾
Key concepts
Compare bar heights side by sideGrouped or stacked bars show two variablesTallest/shortest bar firstCategory order can be arbitrary
Video resources
Khan AcademyReading bar graphs↗
Test tip: If a question asks for the difference between two bars, read both exact values off the axis first, then subtract. Don't try to estimate the gap by eye.
Line graphs — change over time▾
Key concepts
Each point is one measurementSlope shows rate of changeSteeper line = faster changeFlat line = no changePeaks & valleys mark turning points
Video resources
Khan AcademyReading line graphs↗
Test tip: When a question asks where change happened "the most," compare the steepness between each pair of consecutive points — don't just compare the highest point to the lowest one.
Pie charts — parts of a whole▾
Key concepts
All slices add up to 100%Combine slices for "combined" questionsBigger slice ≠ most important detailLegend matches color to category
Video resources
Khan AcademyReading pie graphs (circle graphs)↗
Test tip: If a question asks about two categories combined, just add their percentages directly from the legend — there's no need to estimate slice sizes visually.
Data tables — exact values, careful matching▾
Key concepts
Match column header AND row headerWatch for unit changes between columnsDon't drift to the wrong rowTotals may need to be calculated
Test tip: Trace your finger along the row and down the column at the same time to land on the exact cell. Most table mistakes come from drifting to a neighboring row, not from misunderstanding what's being asked.
Diagrams, maps & political cartoons▾
Key concepts
Diagrams: trace arrows/labels in orderMaps: check the legend/key firstCartoons: labels, exaggeration, symbolismCaptions often state the point directly
Resources
Library of CongressPolitical cartoons — finding point of view↗
Test tip: For a political cartoon, identify every labeled object first, then ask what's exaggerated and why. The combination of labels plus exaggeration is almost always where the cartoonist's point lives.
3
Common Traps on Test Day
Two reasoning mistakes the test is specifically designed to catch — both come up across science and social studies passages alike.
Correlation isn't causation▾
Key concepts
Two trends together ≠ one causes the other"Affects" claims need more than a patternCheck what the choice actually claims
Test tip: If an answer choice says one variable "causes" or "leads to" another based only on a graph showing both rising or falling together, treat it with suspicion — a graph by itself usually shows correlation, not proof of cause.
Percentages vs. raw numbers▾
Key concepts
Bigger percent ≠ bigger raw countConfirm what the question is asking forCombining percentages needs the same wholeSample size changes what a percent means
Test tip: Before calculating anything, reread the question to confirm whether it wants a percentage or an actual count. These two answers can look similar in a graph but are rarely interchangeable.
